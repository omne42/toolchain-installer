# 架构设计

## 目标

把 `git` / `gh` 的获取与安装做成独立可复用能力，供多个调用方共享，并保持调用方无关。

## 组件分层

1. `installer-cli`  
统一入口，负责参数解析、执行流程编排、JSON 输出。

2. `catalog`  
维护工具来源定义（官方源、公共镜像、平台资产匹配规则）。

3. `resolver`  
基于 `tool/version/os/arch` 解析下载候选序列，并执行可达性探测与优先级排序。

4. `fetcher + verifier`  
下载资产并完成完整性校验（哈希/签名元信息）。

5. `installer`  
按平台落盘：解压、权限设置、原子替换、可执行性检查。

6. `gateway`（可选）  
Cloudflare Worker 固定路由重定向层，用于网络优化与滥用防护。

7. `plan executor`
执行调用方给定的安装计划（`release/system_package/pip`），由调用方决定安装对象。

## 边界约束

- 不耦合 `omne-agent` 私有协议、事件或目录结构。
- 不把业务逻辑写进调用方（调用方只消费 CLI 契约）。
- 不提供任意 URL 代理，不承接非白名单下载。
- 不托管私有二进制分发服务。

## 与调用方关系

- `omne-agent`：通过子进程调用 CLI，解析 JSON 结果映射到自身 `toolchain bootstrap` 输出。
- 其他调用方：可直接调用 CLI 或封装 SDK 适配层，协议保持一致。

## 验收标准

- 新增/替换调用方时，无需修改安装器核心逻辑。
- 调用方仅依赖契约字段，不依赖日志文本。
- 关闭 `gateway` 时，安装器仍可直接走公共源工作。
