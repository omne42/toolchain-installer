# 安全与反滥用

## 威胁模型

- 滥用网关作为开放代理转发任意流量。
- 高频请求消耗网关配额或触发上游限流。
- 篡改下载内容导致执行恶意二进制。
- 通过异常参数触发 SSRF 或白名单逃逸。

## 强制策略

1. 固定路由  
仅允许 `/toolchain/{tool}/{version}/{os}/{arch}`，禁止 `?url=` 任意目标参数。

2. 白名单来源  
只允许预定义域名与资产模式，非白名单直接拒绝。

3. 中国流量限定  
Worker 仅允许 `CF-IPCountry=CN` 的请求，其他国家/地区直接拒绝。

4. git release 限定  
Worker 仅服务 `git release` 资产路由，不代理 `gh` 或其他工具下载。

5. 默认重定向  
网关优先返回 `302/307`，不转发大文件内容。

6. 限流  
按 IP + 路径限流，超过阈值返回 `429`。

7. 完整性校验  
下载后必须进行哈希校验，校验失败直接终止安装。

8. 最小日志  
日志只记录必要诊断字段，避免泄露凭证或敏感头。

## 验收测试

- 请求 `?url=https://example.com/a` 返回拒绝。
- 非中国流量请求 Worker 返回拒绝。
- `gh` 相关路由请求 Worker 返回拒绝。
- 非白名单域名候选全部被拒绝。
- 压测超阈值后出现稳定 `429`。
- 人工篡改下载文件时，校验失败且不写入目标路径。
