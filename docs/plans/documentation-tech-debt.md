# 文档技术债

## 待处理项

- 自动 Markdown 链接检查
  - 当前只有文档入口存在性检查，还没有覆盖相对链接是否全部可解析。
- 执行计划归档规范
  - 目前已有路线图和债务跟踪，但还没有为复杂改动建立统一的 `active/completed` 计划模板。
- 来源探测执行样例
  - 还缺少官网失效、备用站回退成功时的实际输出样本。
- 外部网关跨仓库接入文档
  - installer 已拆出独立网关项目，但还缺少更完整的跨仓库部署、版本对齐和变更协同说明。
- 调用方接入示例
  - 还缺少 `omne-agent` 之外的调用方接入指南或 SDK 适配示例。

## 处理原则

- 优先补“会导致事实失真或入口失效”的债。
- 其次补“让调用方重复摸索”的债。
- 只在规则稳定后再做更重的自动化，避免把过早抽象固化进 CI。
